Fail2Ban Setup

Create the jail configuration:

# Create jail config
cat > /etc/fail2ban/jail.d/sip-blacklist.conf << 'EOF'
[sip-blacklist]
enabled = true
filter = sip-blacklist
action = iptables-allports[name=sip-blacklist, protocol=all]
logpath = /var/log/freeswitch/freeswitch.log
maxretry = 1
findtime = 86400
bantime = -1
EOF

# Create filter (dummy - we use manual banning)
cat > /etc/fail2ban/filter.d/sip-blacklist.conf << 'EOF'
[Definition]
failregex = ^MANUAL_BAN_ONLY$
ignoreregex =
EOF

# Reload fail2ban
fail2ban-client reload

After setup, the admin panel will add IPs to Fail2Ban automatically when they exceed the threshold.

Note: After making changes, click "Apply & Reload" to reload FreeSWITCH with new ACL rules. Blacklist IPs are always blocked. Whitelist only takes effect when Whitelist Mode is enabled. Fail2Ban provides permanent firewall-level blocking for persistent attackers.
